# 訓練

## ds004504_rbp_paper 目標

`ds004504_rbp_paper` 的目標是重現 paper 的流程與結果，不是改良 paper。

因此，這條 reproduction path 會盡量照 paper protocol 做。即使 paper 流程存在明顯方法學問題，例如 normalization 或 split 造成資料洩漏，只要那是 paper 流程的一部分，`ds004504_rbp_paper` 也應該照做並清楚標記。

如果之後要做更嚴謹的版本，例如 subject-level split、train-only normalization、或重新設計 label/task protocol，應該建立新的 experiment 名稱，不要混進 `ds004504_rbp_paper`。

## 程式位置

| 位置 | 用途 |
|---|---|
| `cfgs/ds004504_rbp_paper/base.yaml` | paper reproduction 的共同 dataset artifact path、processing command 參數、training/runtime 設定 |
| `cfgs/ds004504_rbp_paper/*.yaml` | 各分類任務的 YAML config，透過 `extends` 覆蓋 base |
| `src/ecg/training/loop.py` | 通用 PyTorch train/eval epoch loop，不知道 dataset |
| `src/ecg/training/metrics.py` | 通用分類 metrics，不知道 ds004504 label 語意 |
| `src/ecg/training/ds004504_rbp_paper/tasks.py` | paper 四個分類任務的 label filter/remap |
| `src/ecg/training/ds004504_rbp_paper/normalization.py` | paper-style full-task min-max normalization |
| `src/ecg/training/ds004504_rbp_paper/splits.py` | paper-style epoch-level stratified split |
| `src/ecg/training/ds004504_rbp_paper/factory.py` | 組裝 ds004504_rbp_paper Dataset、normalization、split、model、DataLoader |
| `scripts/ds004504_rbp_paper/train.py` | 讀取單一 YAML path 並執行 paper reproduction training |

## Paper Reproduction Protocol

目前 `ds004504_rbp_paper` training 實作使用：

- input feature: `X_rbp_mean`
- input shape: `[n_bands, 1]`
- paper-style min-max normalization: 對 selected task dataset fit，再 split
- split: epoch-level stratified，比例由 YAML 的 `split.train_fraction`、`split.val_fraction`、`split.test_fraction` 指定
- model: TCN-LSTM
- TCN filters: 32
- TCN kernel size: 7
- TCN dilations: `(1, 1)`
- TCN dropout: 0.3
- LSTM hidden size: 64
- dense hidden sizes: `(128, 192, 256)`
- dense dropout: 0.2
- batch size: 32
- optimizer: Adam
- learning rate: 0.0001

這些值屬於 `ds004504_rbp_paper` reproduction contract，不是通用模型 default。

## Training Scripts

三分類：

```bash
uv run python scripts/ds004504_rbp_paper/train.py cfgs/ds004504_rbp_paper/multiclass.yaml
```

AD + FTD vs Healthy：

```bash
uv run python scripts/ds004504_rbp_paper/train.py cfgs/ds004504_rbp_paper/ad_ftd_vs_healthy.yaml
```

AD vs Healthy：

```bash
uv run python scripts/ds004504_rbp_paper/train.py cfgs/ds004504_rbp_paper/ad_vs_healthy.yaml
```

FTD vs Healthy：

```bash
uv run python scripts/ds004504_rbp_paper/train.py cfgs/ds004504_rbp_paper/ftd_vs_healthy.yaml
```

`scripts/ds004504_rbp_paper/train.py` 只接收一個 positional argument：YAML config path。`epochs`、`seed`、`device`、`num_workers`、`split`、`evaluation.test_source`、`output.dir` 等訓練控制參數都必須在 YAML 裡明確指定。

`training.seed: null` 表示 script 會在 runtime 產生實際 seed，並寫入 `run.json` 的 `resolved_seed`；如果需要完全重跑同一組結果，請改成整數 seed。

`split.test_fraction` 和 `evaluation.test_source` 有直接關係：

- `evaluation.test_source: split`：使用獨立 test split 寫出 `test_metrics.json`，因此 `split.test_fraction` 必須大於 0。
- `evaluation.test_source: val`：沒有獨立 test split，最後把 validation split 當作 reported metrics，因此 `split.test_fraction` 必須是 0。
- `evaluation.test_source: none`：只輸出 train/validation history，不寫 `test_metrics.json`，因此 `split.test_fraction` 必須是 0。

`cfgs/ds004504_rbp_paper/val_as_test_80_20/` 是用來驗證 paper 是否把 80/20 validation metrics 當作 reported test metrics 的 scenario。這組 config 使用：

- `split.train_fraction: 0.8`
- `split.val_fraction: 0.2`
- `split.test_fraction: 0.0`
- `evaluation.test_source: val`

這代表 `test_metrics.json` 名義上仍是 final reported metrics，但來源明確標成 validation split，不是獨立 test split。

YAML merge 規則保持簡單：

- `extends` 是 list，路徑相對於目前 YAML。
- parent YAML 依序讀取。
- dict 做 recursive merge。
- scalar 與 list 直接覆蓋。
- CLI 不提供 override。

## Outputs

每次訓練會在 `--output-dir` 寫出：

| 檔案 | 說明 |
|---|---|
| `run.json` | task、路徑、requested/resolved seed、paper hyperparameters、normalization params、split fractions、split indices、evaluation source |
| `history.json` | 每個 epoch 的 train/validation loss 與 metrics |
| `test_metrics.json` | final model 在 `evaluation.test_source` 指定資料上的 loss 與 metrics；`test_source: none` 時不會產生 |
| `test_predictions.csv` | `test_metrics.json` 對應資料的 prediction rows；`test_source: none` 時不會產生 |
| `model.pt` | PyTorch model state dict 與重建模型所需 metadata |
