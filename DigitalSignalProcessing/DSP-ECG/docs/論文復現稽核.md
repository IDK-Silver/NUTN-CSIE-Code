## 結論

這篇論文不能只照文字描述直接嚴格復現。PDF 內部有至少三個會影響復現判斷的問題：

1. 論文寫 `80%/10%/10%` train/validation/test，但結果表與混淆矩陣的 support 接近 `80%/20%` holdout，而不是 10% test。
2. 多分類與 FTD vs Healthy 結果中，Frontotemporal 與 Healthy 的 support 跟 OpenNeuro `participants.json` 定義不一致，強烈暗示部分圖表的 FTD/Healthy 標籤或 support 被交換。
3. Table 6 的 Healthy support 與 Figure 4d、文字敘述互相矛盾。

因此目前比較合理的判斷是：論文的結果區至少有表格/標籤錯誤；若要評估方法本身，必須分成「照論文文字復現」、「照論文表格 support 反推復現」與「修正資料洩漏後的 subject-wise 復現」三套實驗。

## 論文本身的可疑錯誤

### 1. Split 描述與結果 support 不一致

PDF Section 3.4 說資料分成 `80%/10%/10%`。但 PDF 結果表的 support 大約是每類 20%：

| 任務 | PDF support | 若是 10% test 應接近 | 現有 H5 10% stratified test |
|---|---:|---:|---:|
| AD, multiclass | 1,876 | 約 966 | 967 |
| FTD, multiclass | 1,597 | 約 550 | 551 |
| Healthy, multiclass | 1,106 | 約 800 | 800 |

現有 H5 總 epoch 數為 23,151；10% test 不可能產生 PDF 中約 4,579 筆的 multiclass support。這比較像 80/20 split 或把 validation 當 reported test。

### 2. FTD 與 Healthy label/support 疑似交換

OpenNeuro `participants.json` 明確定義：

- `A`: Alzheimer Disease Group
- `F`: Frontotemporal Dementia Group
- `C`: Healthy Group

現有處理後 epoch counts：

| Group | Meaning | Epochs |
|---|---|---:|
| A | Alzheimer | 9,659 |
| F | Frontotemporal Dementia | 5,497 |
| C | Healthy | 7,995 |

若用約 20% holdout，FTD 應約 1,100，Healthy 應約 1,600。但 PDF Figure 4a / Table 3 顯示 Frontotemporal support 1,597、Healthy support 1,106，剛好反過來。這不是單純小數四捨五入問題，而是與資料集定義相反。

### 3. Table 6 與 Figure 4d 矛盾

PDF Figure 4d 的 FTD vs Healthy confusion matrix 是：

```text
Frontotemporal: 1590 correct, 7 misclassified, support 1597
Healthy:          1 misclassified, 1105 correct, support 1106
```

但 PDF Table 6 寫 Healthy support `1596`。若用 Figure 4d 的矩陣計算，accuracy 是 `(1590 + 1105) / (1597 + 1106) = 0.9970`，正好等於 Table 7；所以 Table 6 的 Healthy support 很可能是 typo，應是 `1106`。同時，這仍保留 FTD/Healthy support 交換的問題。

## 現有 repo 復現狀態

目前 repo 的資料處理已接近論文 modified RBP：

- 使用 derivatives `.set`
- 6 秒 epoch、50% overlap
- six-band modified RBP
- 跨 channel 平均成 `X_rbp_mean`

但現有已產生 runs 沒有復現論文 accuracy：

| 任務 | PDF accuracy | 現有 run test accuracy |
|---|---:|---:|
| AD vs FTD vs Healthy | 0.8034 | 0.5720 |
| AD + FTD vs Healthy | 0.9980 | 0.7467 |
| AD vs Healthy | 0.9974 | 0.7368 |
| FTD vs Healthy | 0.9970 | 0.7424 |

這些既有 runs 的 `test_predictions.csv` 仍是舊格式，只有 `index,y_true,y_pred`，不是 README/task 中描述的新欄位格式。若要做完整報告 CSV 與 ROC/AUC，四個任務需要重跑。

## 方法學問題

即使不考慮論文表格錯誤，論文方法本身也有嚴重高估風險：

- 以 epoch 為單位隨機切分，沒有說明 subject-wise split。同一受試者甚至相鄰 50% overlap epoch 可能同時出現在 train/test。
- 論文 Section 3.4 的流程是先 normalize 再 split；repo 的 paper-style factory 也先對整個 task features fit min-max，再切 split。這會讓 test distribution 進入 normalization statistics。
- 二元分類 99.7% 以上 accuracy 在這種 split 設定下不能視為對 unseen subject 的臨床泛化能力。

## 建議的復現路線

1. `paper_literal_80_10_10`：照 PDF 文字，OpenNeuro `v1.0.5`、modified RBP、80/10/10 epoch split、重跑四任務。
2. `paper_table_inferred_80_20`：照 PDF support 反推，80/20 epoch split，把 validation 當 reported test，檢查是否可接近 Figure 4 / Tables 3-7。
3. `label_audit`：固定 split 後同時用正確 label mapping 與 F/C swapped mapping 跑一次，驗證 PDF 的 support 是否來自標籤交換。
4. `subject_wise_corrected`：以 subject 分層切分，normalization 只 fit train，再 transform val/test，作為可信泛化結果。
5. 報告時分開列出 paper-claimed、paper-inferred、corrected protocol，不要把 paper-inferred 結果稱為嚴格復現。
