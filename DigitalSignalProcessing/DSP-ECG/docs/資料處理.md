# 資料處理

## 目標

這個階段只做一件事：把 OpenNeuro `ds004504` 的 BIDS/EEGLAB EEG dataset 轉成後續程式容易讀取的 H5 feature cache。

目前 command 介面是 dataset-aware，呼叫時必須明確指定 `--dataset`。下載資料的 raw dataset id 是 `ds004504`；處理資料的 processed dataset id 是 `ds004504_rbp_paper`。已實作的資料處理 handler 只有 `ds004504_rbp_paper`，其他處理方法可用 `ds004504_{method}` 命名。

`participants.tsv` 的 `Group` 欄位、`derivatives/sub-*/eeg/*_task-eyesclosed_eeg.set` 路徑、label mapping、RBP bands、epoch/Welch 參數，以及輸出的 RBP H5 schema 都由 `ds004504_rbp_paper` handler 負責。`cfgs/recipes/ds004504_rbp_paper.yaml` 保存 paper-style RBP cache 的 command 參數組合，但 command 不會自動讀取 recipe。

這個階段不做：

- train / validation / test split
- normalization
- PyTorch Dataset
- 模型訓練

## 指令

下載原始資料集：

```bash
uv run download_raw_dataset \
  --dataset ds004504 \
  --tag 1.0.8 \
  --target-dir data/raw/ds004504
```

處理原始資料集並產生 RBP H5：

```bash
uv run process_raw_dataset \
  --dataset ds004504_rbp_paper \
  --raw-dir data/raw/ds004504 \
  --output data/processed_raw_dataset/ds004504_rbp_paper.h5 \
  --manifest data/processed_raw_dataset/ds004504_rbp_paper_manifest.json
```

快速檢查資料路徑：

```bash
uv run process_raw_dataset \
  --dataset ds004504_rbp_paper \
  --raw-dir data/raw/ds004504 \
  --output data/processed_raw_dataset/ds004504_rbp_paper.h5 \
  --manifest data/processed_raw_dataset/ds004504_rbp_paper_manifest.json \
  --dry-run \
  --limit 2
```

只轉一個 subject 做 smoke test：

```bash
uv run process_raw_dataset \
  --dataset ds004504_rbp_paper \
  --raw-dir data/raw/ds004504 \
  --limit 1 \
  --output /tmp/ds004504_rbp_paper_limit1.h5 \
  --manifest /tmp/ds004504_rbp_paper_limit1_manifest.json \
  --overwrite
```

## 程式位置

資料處理相關程式分成以下 package：

| 位置 | 用途 |
|---|---|
| `cfgs/recipes/ds004504_rbp_paper.yaml` | paper-style RBP cache 的顯式 command recipe |
| `src/ecg/commands/` | dataset-aware CLI entrypoints，只負責 parse args 與 dispatch |
| `src/ecg/data/registry.py` | 支援 dataset 的 registry；raw dataset 目前只註冊 `ds004504`，processed dataset 目前只註冊 `ds004504_rbp_paper` |
| `src/ecg/data/pipelines.py` | dataset handler contract 與 processing options |
| `src/ecg/data/raw_ds004504.py` | OpenNeuro `ds004504` raw dataset 下載 handler |
| `src/ecg/data/ds004504_rbp_paper.py` | `ds004504_rbp_paper` processed dataset handler：讀取、label mapping、H5/manifest 寫入 |
| `src/ecg/data/rbp.py` | RBP 特徵計算 |
| `src/ecg/datasets/` | 後續 PyTorch Dataset 定義 |
| `src/ecg/models/` | 後續 model 定義 |

## 輸入與輸出

輸入資料：

```text
data/raw/ds004504
```

主要輸出：

```text
data/processed_raw_dataset/ds004504_rbp_paper.h5
data/processed_raw_dataset/ds004504_rbp_paper_manifest.json
```

`ds004504_rbp_paper.h5` 是完整 processed feature cache。它保存全部 epoch，不分 train / val / test。

`ds004504_rbp_paper_manifest.json` 是處理紀錄，應記錄來源資料、處理參數、label mapping、subject/epoch 數量等資訊，方便之後追溯。

## H5 Schema

`ds004504_rbp_paper.h5` 目前包含以下 datasets：

| 欄位 | dtype | shape | 說明 |
|---|---|---|---|
| `X_rbp_channel` | `float32` | `[n_epochs, n_channels, n_bands]` | 每個 epoch、每個 channel、每個 frequency band 的 RBP |
| `X_rbp_mean` | `float32` | `[n_epochs, n_bands]` | 由 `X_rbp_channel.mean(axis=1)` 算出，跨 channel 平均後的 RBP |
| `y` | `int64` | `[n_epochs]` | 從 `participants.tsv` 的 `Group` 編碼而來的 class label |
| `subject_id` | string | `[n_epochs]` | 每個 epoch 對應的 BIDS participant id，例如 `sub-001` |
| `epoch_start_sec` | `float32` | `[n_epochs]` | 每個 epoch 在原始 EEG recording 中的開始秒數 |
| `channel_names` | string | `[n_channels]` | EEG channel 名稱 |
| `band_names` | string | `[n_bands]` | RBP band 名稱 |

## RBP 欄位來源

`X_rbp_channel` 是主要保留的 RBP 特徵，數值來源如下：

```text
EEG .set
  -> 取單一 channel 的 epoch 訊號
  -> Welch method 算 PSD
  -> 依照 frequency band 加總 PSD bins
  -> band power / 0.5-45 Hz total power
  -> 得到該 channel 的 RBP
```

因此：

```text
X_rbp_channel[epoch, channel, band]
```

代表某個 epoch 中，某個 EEG channel 在某個 frequency band 的相對功率。
`zaeta` band 名稱沿用論文拼法 `Zaeta`，範圍是 16-24 Hz，不改成 `zeta`。

`X_rbp_mean` 可以從 `X_rbp_channel` 算出：

```python
X_rbp_mean = X_rbp_channel.mean(axis=1)
```

其中 `axis=1` 是 channel 維度。

## Label Mapping

`y` 來自 `participants.tsv` 的 `Group` 欄位：

| Group | 類別 |
|---|---|
| `A` | Alzheimer |
| `F` | Frontotemporal dementia |
| `C` | Healthy control |

實際數值編碼應以 `ds004504_rbp_paper_manifest.json` 裡記錄的 `label_map` 為準。
