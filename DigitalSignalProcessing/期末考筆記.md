## DFT 與 FFT

DFT（Discrete Fourier Transform）把有限長度的離散時間訊號轉成離散頻率成分，讓我們知道訊號中有哪些頻率、各頻率強度多大。對音訊來說，DFT 可以把時間波形轉成頻譜，用來找出鋼琴聲中的基頻與諧波。

FFT（Fast Fourier Transform）不是新的轉換，而是快速計算 DFT 的演算法。DFT 直接計算很慢，FFT 利用分治與對稱性降低運算量，因此適合即時或大量音訊分析。

問答題句型：

> DFT 的用途是把離散時間訊號轉到頻域，FFT 則是有效率地計算 DFT。鋼琴音高辨識會用 FFT 找頻譜峰值，再推估音符。

## 頻率解析度

頻率解析度代表頻譜中相鄰 bin 的頻率間距：

```text
頻率解析度 = 取樣率 / FFT 點數
```

解析度越小，越容易分辨相近的音高；但要得到更細解析度，通常需要更長的分析窗。長窗會讓時間定位變差，因此在鋼琴音訊中要在「音高解析度」與「起音時間準確度」之間取捨。

問答題重點：

- FFT 點數越大，頻率 bin 越密。
- 取樣率固定時，長 window 能提升頻率解析度。
- 短 window 能看清楚音什麼時候開始，但頻率解析度較差。
- 鋼琴辨識需要同時考慮音高與起音，因此常用 STFT 做折衷。

## Spectral Leakage 與 Window

Spectral leakage 是指原本集中在某個頻率的能量，因為截取有限長度訊號而擴散到附近頻率 bin。若直接切一段音訊做 FFT，切斷點通常不會剛好落在週期邊界，會造成頻譜外洩。

Window function 的作用是在分析片段兩端做平滑，降低突然截斷造成的 leakage。常見 window 包含 rectangular、Hamming、Hann 等。

答題方向：

- Rectangular window 頻率主瓣窄，但旁瓣大，leakage 較明顯。
- Hann/Hamming window 可降低旁瓣與 leakage，但主瓣會變寬。
- 在音高辨識中，window 能讓頻譜峰值更穩定，但也可能犧牲部分頻率解析度。

## STFT 與 Spectrogram

STFT（Short-Time Fourier Transform）把長音訊切成一段段短 frame，每段做 FFT，得到隨時間變化的頻譜。Spectrogram 則是把 STFT 結果視覺化，橫軸是時間，縱軸是頻率，顏色或亮度代表能量。

鋼琴聲不是永遠不變的訊號。每個音都有 attack、decay、sustain、release，因此只看整段 FFT 會失去時間資訊。STFT 可以觀察每個時間點有哪些頻率成分，適合用於起音偵測與音高追蹤。

重要取捨：

| 參數 | 太小 | 太大 |
|---|---|---|
| Window length | 頻率解析度差 | 時間定位差 |
| Hop size | 運算量大、frame 多 | 時間解析度差，可能漏掉短音 |
| Overlap | 結果較平滑但較耗時 | 結果較粗糙但較快 |

## 數位濾波

數位濾波器用來保留想要的頻率成分，抑制不想要的頻率成分。

| 濾波器 | 用途 |
|---|---|
| Low-pass | 保留低頻，移除高頻雜訊 |
| High-pass | 移除 DC offset、低頻震動或環境雜訊 |
| Band-pass | 只保留某段頻率，適合聚焦在樂器音域 |
| Notch | 移除特定頻率，例如電源 hum |

FIR 與 IIR 比較：

| 項目 | FIR | IIR |
|---|---|---|
| 結構 | 只用目前與過去輸入 | 使用回授 |
| 穩定性 | 較容易保證穩定 | 設計不當可能不穩定 |
| 相位 | 可設計成線性相位 | 通常非線性相位 |
| 運算 | 通常階數較高 | 通常較省係數 |

問答題句型：

> FIR 適合需要線性相位與穩定性的場合；IIR 可用較低階數達到尖銳頻率響應，但要注意穩定性與相位失真。

## 鋼琴音訊專案流程

鋼琴音高辨識可以整理成以下流程：

```text
錄音或讀入音訊
  -> 轉 mono、正規化、去除靜音或雜訊
  -> 分 frame 或做 STFT
  -> 找起音位置或能量變化
  -> 對每個 frame 做 FFT / 自相關 / HPS
  -> 找基頻 f0
  -> 把 f0 轉成音名或 MIDI note
  -> 用時間平滑與門檻修正結果
```

每一步的目的：

- 轉 mono：降低資料量，避免左右聲道差異影響分析。
- 正規化：讓不同錄音音量可比較。
- 分 frame：讓模型或演算法能追蹤音高隨時間的變化。
- 起音偵測：找出鋼琴鍵被敲下的時間點。
- 頻譜分析：找出基頻與諧波。
- 音符對應：把頻率轉成鋼琴鍵或音名。

## 音高、基頻與諧波

鋼琴每個音的音高主要由基頻決定，但實際鋼琴聲不是單一正弦波，而是由基頻加上多個諧波組成：

```text
基頻 f0
第 2 諧波 2f0
第 3 諧波 3f0
第 4 諧波 4f0
...
```

鋼琴音色來自這些諧波的能量比例與時間包絡。不同樂器即使彈同一音高，基頻相同，但諧波分布不同，所以聽起來音色不同。

答題重點：

- 基頻決定音高。
- 諧波決定音色的重要部分。
- FFT 頻譜中的最大峰值不一定是基頻，因為某些諧波可能比基頻強。
- 因此鋼琴音高辨識不能只盲目選最大峰值。

## 音符與頻率對應

十二平均律中，相鄰半音頻率比固定為：

```text
2^(1/12)
```

A4 通常定義為 440 Hz。若知道頻率，可以用下面概念轉成 MIDI note：

```text
MIDI note = 69 + 12 * log2(f / 440)
```

問答題不需要手算，但要知道：

- 八度音頻率相差 2 倍。
- 半音之間是等比關係，不是等差。
- A4 = 440 Hz 是常用調音基準。
- 頻率估計後要找最接近的標準音符。

常見參考：

| 音名 | 約略頻率 |
|---|---:|
| C4 | 261.63 Hz |
| D4 | 293.66 Hz |
| E4 | 329.63 Hz |
| F4 | 349.23 Hz |
| G4 | 392.00 Hz |
| A4 | 440.00 Hz |
| B4 | 493.88 Hz |
| C5 | 523.25 Hz |

## 起音偵測

起音（onset）是音符開始的時間點。鋼琴的 attack 很明顯，因此可以用能量變化或頻譜變化偵測。

常見方法：

- Energy-based：看短時間能量是否突然上升。
- Spectral flux：看相鄰 frame 的頻譜變化量。
- Thresholding：設定門檻，超過才判斷為新音。
- Peak picking：在 onset strength 曲線上找局部峰值。

答題重點：

> 音高辨識只知道「是哪個音」，起音偵測則知道「音什麼時候開始」。鋼琴專案若要辨識旋律或琴鍵序列，兩者都需要。

## 為什麼鋼琴辨識不簡單

常見困難：

- 基頻可能不是最大峰值。
- 低音區諧波密集，容易混淆。
- 高音音長短，能量衰減快。
- 和弦是多個音同時出現，頻譜會重疊。
- 延音踏板會讓前一個音殘留，干擾下一個音。
- 錄音環境有雜訊、混響與麥克風頻率響應差異。
- Window 太短頻率不準，太長又會模糊起音時間。

可以改善的方法：

- 使用適當 window 與 overlap。
- 不只看最大峰值，也檢查諧波結構。
- 使用 autocorrelation 或 harmonic product spectrum 推估基頻。
- 對估計結果做時間平滑。
- 用 onset detection 把不同音符切開。
- 限定鋼琴音域，過濾不合理頻率。

## Autocorrelation 與 HPS

Autocorrelation 的想法是把訊號和自己延遲後的版本比較。如果某個延遲量讓波形高度相似，就代表訊號可能有對應的週期。週期的倒數就是基頻。

HPS（Harmonic Product Spectrum）的想法是利用諧波關係。把頻譜壓縮成 1/2、1/3、1/4 倍後相乘，真正的基頻位置會因為多個諧波對齊而被強化。

比較：

| 方法 | 優點 | 限制 |
|---|---|---|
| FFT peak picking | 簡單直觀 | 容易把強諧波當基頻 |
| Autocorrelation | 適合找週期 | 對多音、雜訊、短 frame 較敏感 |
| HPS | 可利用諧波結構找基頻 | 對頻譜解析度與諧波完整度敏感 |

## 問答題題庫

### 1. DFT 和 FFT 的差別是什麼？

DFT 是把離散時間訊號轉成離散頻率成分的數學轉換；FFT 是快速計算 DFT 的演算法。鋼琴音訊分析會用 FFT 取得頻譜，進一步找基頻與諧波。

### 2. 為什麼音訊分析需要頻域？

時間波形可以看出振幅隨時間變化，但不容易直接知道有哪些音高。頻域能顯示不同頻率的能量分布，適合分析音高、諧波、噪聲和濾波效果。

### 3. 什麼是 spectral leakage？如何降低？

Spectral leakage 是有限長度截取造成頻率能量擴散到附近 bin 的現象。可以使用 Hann 或 Hamming window 平滑截取邊界，降低旁瓣能量與外洩。

### 4. STFT 為什麼比整段 FFT 更適合鋼琴音訊？

鋼琴音會隨時間開始、衰減、消失。整段 FFT 只給整體頻譜，無法知道音符什麼時候出現；STFT 保留時間與頻率資訊，因此能做音高追蹤與起音偵測。

### 5. Window length 怎麼影響結果？

長 window 頻率解析度較好，但時間解析度較差；短 window 能更準確定位起音，但頻率解析度較差。鋼琴辨識要在音高準確度和起音時間準確度之間折衷。

### 6. 為什麼最大頻譜峰值不一定是鋼琴音的基頻？

鋼琴聲包含基頻和多個諧波，某些諧波能量可能比基頻更強。如果只取最大峰值，可能把 2f0 或 3f0 誤判成真正音高。

### 7. 如何把頻率轉成音符？

十二平均律中相鄰半音的頻率比為 2^(1/12)，A4 定義為 440 Hz。估計出基頻後，可以找最接近的標準音符或 MIDI note。

### 8. 為什麼鋼琴專案需要起音偵測？

音高估計只能知道某段 frame 的主要頻率，起音偵測可以找出每個音符開始的時間。若要辨識旋律、彈奏順序或切分每個琴鍵事件，就需要起音偵測。

### 9. FIR 和 IIR 濾波器差在哪裡？

FIR 沒有回授，較容易保證穩定，也可設計線性相位；IIR 有回授，可用較低階數達到尖銳頻率響應，但要注意穩定性與相位失真。

### 10. 鋼琴辨識中濾波可以做什麼？

濾波可以移除低頻震動、高頻雜訊或非鋼琴音域的成分，也可以聚焦在鋼琴主要頻率範圍，使後續 FFT、起音偵測或基頻估計更穩定。

### 11. 為什麼和弦比單音難辨識？

和弦包含多個基頻與各自諧波，頻譜成分會重疊。某個音的諧波可能落在另一個音的基頻附近，導致單純 peak picking 難以分辨。

### 12. 若要改善鋼琴音高辨識準確率，可以怎麼做？

可以使用 STFT、適當 window、onset detection、諧波結構檢查、autocorrelation、HPS、時間平滑，以及限制合理鋼琴音域。若是和弦辨識，還需要更進階的多音高估計方法。

### 解釋一個 DSP 方法

```text
這個方法的目的，是把原始時間訊號轉成比較容易分析的形式。
在鋼琴音訊中，它可以幫助我們找出音高、起音或雜訊成分。

```

### 解釋鋼琴專案流程

```text
先讀入鋼琴音訊並做基本前處理，例如轉 mono、正規化與濾波。
接著用 STFT/FFT 做頻譜分析，配合 onset detection 找出每個音符事件。
對每個事件估計基頻，再用十二平均律把頻率對應到音名或 MIDI note。
最後用時間平滑、門檻與音域限制修正誤判。
```

### 解釋限制

```text
這個方法在單音鋼琴上較直觀，但遇到和弦、延音踏板、雜訊、混響或基頻較弱時容易失誤。
主要原因是頻譜峰值不一定等於基頻，而且多個音的諧波會互相重疊。
```
